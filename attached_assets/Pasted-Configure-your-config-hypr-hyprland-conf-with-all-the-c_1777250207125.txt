1. The “Golden List” — Key Components for Custom Theming in Hyprland
To theme everything for a cohesive and beautiful Hyprland desktop, you’ll want to know which components control what part of the look and feel. Here’s your golden checklist for full theme coverage:

Core Components and Theme Files
Component	Config/Theme File Location	What It Controls
Hyprland	~/.config/hypr/hyprland.conf	Window manager keybinds, rules, wallpaper, window decorations (limited, most visuals from clients & bar)
Waybar	~/.config/waybar/config, ~/.config/waybar/style.css	Status/task bar (top/bottom bar), workspace, modules, colors
Alacritty	~/.config/alacritty/alacritty.yml	Terminal colors, font, transparency
Wofi	~/.config/wofi/config, ~/.config/wofi/style.css	App launcher theme/colors
Mako	~/.config/mako/config	Notifications style, border, color
GTK Apps	~/.config/gtk-3.0/settings.ini,[GTK theme dir]	File manager (Thunar), dialogs, etc.
QT Apps	~/.config/qt5ct/qt5ct.conf, [Qt theme]	For QT apps if you use them
Cursors	~/.icons/default/index.theme, ~/.config/hypr/hyprland.conf	Mouse pointer style
Wallpapers	Any path, set in Hyprland config (wallpaper = ...)	Background
Extra “Look” Stuff
Fonts: Set in each app config and in Waybar
Icons: ~/.icons (for GTK/QT apps and file managers)
Compositor: Hyprland handles transparency, but some people use picom for extra effects (not needed here)

3. How to Safely Add or Combine Themes
Back up your config before overwriting anything:
bash
cp -r ~/.config/waybar ~/.config/waybar.bak
Mix and match:
After downloading a new config, open both yours and the new one, and combine pieces you like.
Restart the component for changes to take effect (waybar: pkill waybar && waybar &, mako: makoctl reload).
