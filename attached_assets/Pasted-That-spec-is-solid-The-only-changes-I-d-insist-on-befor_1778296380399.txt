Implementation notes (the key pitfalls)
A) Correct “real user” output dir & logs under sudo
When you run sudo bash /tmp/nyxus-build-iso.sh --yes:

$HOME becomes /root
~ expands to /root
So instead do something like:

REAL_USER="${SUDO_USER:-$(id -un)}"
REAL_HOME="$(getent passwd "$REAL_USER" | cut -d: -f6)"
Then default:

OUT="${OUT:-$REAL_HOME/nyx-out}"
log file goes there too.
B) Loop devices check (actually test, don’t just look)
A reliable check:

bash
modprobe loop 2>/dev/null || true
[[ -e /dev/loop-control ]] || die "No /dev/loop-control (loop module unavailable)."
losetup -f >/dev/null 2>&1 || die "losetup cannot allocate a loop device."
C) Disk space check should be on the filesystem that backs the workdir
If workdir=/var/tmp/..., check free space on /var/tmp (not / generically):

bash
avail_kb=$(df -Pk "$WORKDIR_BASE" | awk 'NR==2{print $4}')
D) mkarchiso exit codes + grep checks
Do both:

fail if mkarchiso exits non-zero
and fail if grep finds known-bad patterns (your two strings)
Also consider adding these two additional “bad patterns” that often indicate a doomed ISO:

Initcpio image generation failed
Target not found (pacman package missing)
E) “Proceed?” prompt
Make it print a clear plan (as you described) and require explicit y unless --yes.
