Build a native GTK4 Python application and background service 
called NYXUS SAGE.

NYXUS SAGE is an intelligent code auditing system, knowledge 
base, and passive learning engine. It audits any code, script, 
config, or build — learns from everything it sees — and gets 
smarter every single day automatically.

DO NOT design colors, theme, or NyX brand styling — that will
be applied separately. Focus entirely on functionality.
Hand drawn hand written visual style throughout — same as all
other NYXUS apps.

ARCHITECTURE OVERVIEW:
NYXUS SAGE has four connected components:
1. Knowledge Base — ChromaDB vector database storing all 
   knowledge, docs, rules, and learned patterns
2. Audit Engine — scans any file or folder and produces 
   a detailed report
3. Terminal Watcher — silent background service that learns
   passively from everything the user does
4. GTK4 App — visual interface for auditing, reports, 
   rule management, and knowledge base browsing

ALL FOUR must be built and working together.

GTK4 specific rules — CRITICAL:
- Action names must only contain letters numbers hyphens dots
  No spaces parentheses or special characters anywhere
- Never import CSS resources that dont exist
- All UI updates must happen on main thread only
- Background work must use GLib.timeout_add or threads
- Threads must never directly modify GTK widgets
- Always use GLib.idle_add to update UI from threads
- Polling threads must have minimum time.sleep(2)
- Never use tight loops for monitoring
- VTE4 clipboard must use wl-copy for Wayland
- Always handle widget destroy signals
- CSS class names must be valid identifiers
- All signal connections must have matching disconnections
- Window close must not exit app if tray icon present
- All file paths must use os.path or pathlib never hardcoded
- Config must go to ~/.config/appname/ never elsewhere
- Every button must be connected to a real handler
- No placeholder lambda functions on buttons
- All dialogs must have parent window set

Bash and Zsh rules:
- Always quote variables — $var should be "$var"
- Use [[ ]] not [ ] for conditionals in bash
- Always check command exit codes
- Use set -e set -u set -o pipefail in scripts
- Never use ls in scripts — use glob patterns
- Avoid parsing ls output
- Use $() not backticks for command substitution
- Always handle signals in long running scripts
- Heredoc formatting rules
- Function naming conventions
- Local variable declarations in functions
- Script must have shebang line
- Executable scripts must have correct permissions
- Temporary files must be cleaned up

Security rules:

COMPONENT 2 — AUDIT ENGINE:

Command line interface:
nyxus-audit <file or folder> — audit anything
nyxus-audit <file> --fix — auto fix simple issues
nyxus-audit <folder> --report — generate full PDF report
nyxus-audit --learn <file> — add file to knowledge base
nyxus-audit --rules — list all active rules
nyxus-audit --add-rule — interactive rule addition

Audit process:
1. Scan all files in target — detect file types automatically
2. Run all applicable rule collections against each file
3. Cross-reference patterns against knowledge base
4. Check NYXUS specific Layer 4 rules
5. Check for security vulnerabilities
6. Check for common bugs and anti-patterns
7. Generate severity rated report

Severity levels:
CRITICAL — will cause crash or security breach fix immediately
HIGH — likely to cause bugs or failures fix before shipping
MEDIUM — bad practice should be fixed
LOW — style or convention issue
INFO — suggestion for improvement

Report format:
NYXUS SAGE AUDIT REPORT
========================
Target: ~/Projects/GodsApp/
Files scanned: 47
Total issues: 23
Scan time: 2.3 seconds

CRITICAL (2):
  godsapp.py:234 — GTK4 action name contains spaces
  Action: app.Edit Undo History
  Fix: rename to app.edit-undo-history
  Rule: GTK4-ACTION-001

  scanner.py:891 — Thread missing sleep interval
  Tight loop detected in network_monitor()
  Fix: add time.sleep(2) inside loop
  Rule: GTK4-THREAD-002

HIGH (5):
  [list continues]

AUTO-FIXABLE (8 issues):
  Run: nyxus-audit ~/Projects/GodsApp/ --fix
  to automatically fix 8 of 23 issues

PASS: NO — fix CRITICAL issues before shipping

Auto-fix capability:
Simple issues fixed automatically:
- Add missing copyright headers
- Fix import ordering
- Add missing shebang lines
- Fix file permissions
- Add missing docstrings skeleton
- Fix obvious naming convention violations
- Add missing error handling skeleton
- Fix quoted variables in bash

Learning from audits:
- Every issue found logged to learned_patterns collection
- Every fix applied logged with context
- Patterns that appear frequently get promoted to rules
- User can approve or reject pattern promotions

COMPONENT 3 — TERMINAL WATCHER:

Runs as silent systemd service from boot
Zero UI — completely invisible during normal operation
CPU usage must never exceed 0.3%

What it watches:
- Every command run in terminal — logged with timestamp
- Every file created or modified — content analyzed
- Every error message — pattern extracted and stored
- Every fix applied after an error — fix pattern stored
- Every git commit — code changes analyzed
- Every pip or pacman install — dependency tracked
- Every script executed — analyzed for patterns
- Build processes — success and failure patterns stored

What it learns:
- Your coding patterns — how you structure things
- Your common mistakes — what you often get wrong
- Your fix patterns — how you resolve issues
- Your tool preferences — what commands you use
- Your workflow — order you do things in
- New technologies you adopt — learns them with you
- Errors you encounter and how you fix them

How it stores learning:
- Extracts patterns from everything it sees
- Converts patterns to knowledge base entries
- Tags everything with confidence score
- Low confidence patterns stored but not used for auditing
  until confirmed by repeated observation
- High confidence patterns automatically promoted to rules
- Weekly summary of what was learned shown in SAGE app

Privacy:
- Everything stored locally only
- No data ever leaves the machine
- User can view everything that was logged
- User can delete any logged data
- User can pause learning at any time
- Sensitive data like passwords automatically excluded
  using pattern matching — never stored
