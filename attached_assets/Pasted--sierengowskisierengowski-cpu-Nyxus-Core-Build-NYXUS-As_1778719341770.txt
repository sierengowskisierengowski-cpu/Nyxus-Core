What it is:
NYXUS Assistant is the AI brain of the NyX operating system. It lives inside the OS as a floating overlay panel. It understands natural language, executes real system commands, holds full conversations, listens to your voice, and speaks back. It is the centerpiece feature that makes NyX feel alive and intelligent. Think Siri meets Spotlight meets a terminal meets Claude — but darker, more powerful, more NyX.

Activation:

Ctrl+Space — opens the full NYXUS Assistant overlay panel
Hold Ctrl+Space — activates voice input mode, speak command, release to send
Voice output is toggleable on or off by the user
Escape or clicking outside closes the overlay
Keybind wired in hyprland.conf
Bar icon showing assistant status at all times
Accessible from Spotlight launcher and app menu


Visual Design — Must be Apple and Windows tier or better:
The overlay is a centered floating panel. It must look and feel premium — frosted glass blur behind the panel, subtle animated border glow in the active theme pack color, smooth fade or slide in animation on open. Nothing jarring, nothing basic. The NYXUS name sits at the top of the panel in the brand font. Below is a scrollable conversation history area. At the bottom is the input area — a clean single line text field with a microphone icon on the right that pulses when listening. When voice is active an animated waveform appears. When the assistant is thinking a subtle animated typing indicator shows. Everything is pixel perfect, every spacing, every radius, every shadow, every animation. AI Backend:

Claude API as the default engine
Full conversation context sent with every message — not just the last message, the entire history
System prompt tells the assistant it is NYXUS Assistant, the AI of the NyX operating system, and gives it full knowledge of installed apps, keybinds, available system commands, and NyX features
Assistant has a name and personality consistent with NyX — dark, intelligent, minimal, gets things done, feels like it belongs in this OS. Name: NORA (NyX Operating Response Agent)
Separate command parser layer that intercepts system action requests before or alongside Claude
Action executor layer that actually runs the system commands
Permission system — destructive actions require confirmation before executing
Token management — conversation history truncated intelligently if context gets too long
Rate limiting to prevent API spam
Full error handling for every failure mode — API down, no internet, bad response, command fails
Logging system for debugging
Security layer — all commands sanitized before execution
Fallback: if no internet detected, falls back to Ollama local model if installed, or shows a clear styled offline message
API key stored in ~/.config/nyxus/assistant.json
Conversation history saved to ~/.config/nyxus/assistant-history.json
